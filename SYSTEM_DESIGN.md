# การวิเคราะห์และออกแบบระบบ

## ขอบเขตและผู้ใช้งาน

ระบบรองรับ 3 บทบาท: ผู้เข้ารับการอบรมใช้ลงเวลา ทำแบบทดสอบ ส่ง Learning Space ประเมินและรับเกียรติบัตร; เจ้าหน้าที่ตรวจรายชื่อ บันทึกกรณีพิเศษ ตรวจหลักฐานและออกรายงาน; ผู้ดูแลตั้งค่าระบบ ผู้ใช้ หน่วยอบรม ข้อสอบ QR Code และเกียรติบัตร

กฎผ่านอบรม: Learning Space ผ่าน, ลงเวลาครบ 6 หน่วย, ทำ Pre-test ครบ, Post-test ไม่น้อยกว่า 80%, ประเมินครบ และไม่มีสถานะขาดโดยไม่ได้รับอนุญาต

## Sitemap

```text
พื้นที่ผู้เข้ารับการอบรม
├─ Dashboard / ข้อมูลส่วนตัว
├─ งานที่ต้องทำ
└─ ดาวน์โหลดเกียรติบัตร

พื้นที่เจ้าหน้าที่และผู้ดูแล
├─ Dashboard / ผู้ใช้งาน / หน่วยอบรม
├─ QR Code / ข้อสอบ / ตรวจ Learning Space
├─ รายงานและสถิติ / ส่งออกข้อมูล
└─ ตั้งค่าระบบ / Activity Log / เกียรติบัตร
```

## User Flow หลัก

```text
ยืนยันเลขที่จ่ายตรง → พบรายชื่อ → เลือกกิจกรรม → ตรวจเงื่อนไข → บันทึก → แจ้งผล → อัปเดต Dashboard

สแกน QR → ระบุหน่วย → ตรวจช่วงเวลา+รหัส → ป้องกันรายการซ้ำ → บันทึกเวลา → แจ้งสำเร็จ

เจ้าหน้าที่เข้าสู่ระบบ → ตรวจรายงาน Learning Space → ผ่าน/ขอแก้ไข → คำนวณคุณสมบัติ → ออกเกียรติบัตร → เปิดตรวจสอบสาธารณะ
```

## ฐานข้อมูล

ฐานข้อมูลมี 15 ตารางตามข้อกำหนด: `Users`, `Participants`, `TrainingUnits`, `Attendance`, `Questions`, `PretestResponses`, `PosttestResponses`, `TestScores`, `LearningSpaceReports`, `SatisfactionQuestions`, `SatisfactionResponses`, `Certificates`, `Settings`, `ActivityLogs`, `Notifications` โดยใช้ `PersonId` เชื่อมข้อมูลรายบุคคล, `UnitId` เชื่อมการลงเวลา และใช้ UUID เป็นเลขอ้างอิงรายการเปลี่ยนแปลง
